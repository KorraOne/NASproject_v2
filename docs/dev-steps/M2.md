# M2 — Backend core

**Status:** Not started  
**Validates:** Owner completes first-run setup wizard via API (curl or future UI); `setup_complete` persisted.

## Goal

SQLite metadata store, dashboard admin authentication, setup wizard API, and seeded shared folders (Projects, Invoices, Shared) with no permissions assigned yet.

## Prerequisites

- [M1](M1.md) complete — API running on NUC
- `/var/lib/frogswork/` writable by API service

## Implementation steps

### 1. Database layer

- Add SQLAlchemy or raw SQLite (keep simple — SQLite3 + small wrapper is fine)
- DB path: `/var/lib/frogswork/frogswork.db`
- Models/tables:
  - `dashboard_admin` — bcrypt password hash, created_at
  - `system_settings` — `setup_complete`, timezone, snapshot schedule defaults
  - `shared_folders` — name, path slug, created_at
  - (defer `file_users`, `permissions` to M3/M4)

### 2. Paths module

- [`backend/frogswork_api/paths.py`](../../backend/frogswork_api/paths.py) — constants for `/data`, `/var/lib/frogswork`, share paths

### 3. Setup wizard API (`frogswork_api/auth/`, `frogswork_api/system/`)

| Endpoint | Purpose |
|----------|---------|
| `GET /api/setup/status` | `{ setup_complete: bool }` |
| `POST /api/setup` | Admin password, timezone → create admin, seed folders, mark complete |

Setup `POST` body example: `{ "password": "...", "timezone": "Australia/Sydney" }`

On success:

- Create dashboard admin hash in SQLite
- Create shared folder records + filesystem dirs under `/data/shared/Projects`, etc.
- Set `setup_complete=true`
- **Do not** assign Samba permissions yet (M4)

### 4. Admin session auth

- JWT or signed session cookie after `POST /api/auth/login`
- Middleware/dependency: reject protected routes until setup complete (except setup + health)
- `POST /api/auth/login`, `POST /api/auth/logout`

### 5. Seed folders

| Name | Path |
|------|------|
| Projects | `/data/shared/Projects` |
| Invoices | `/data/shared/Invoices` |
| Shared | `/data/shared/Shared` |

Create directories with correct ownership for future Samba use.

### 6. Tests

- Unit tests: setup flow, auth token, setup_complete guard
- Use temp SQLite file in tests

### 7. Update docs

- [`docs/api.md`](../api.md) — document setup and auth endpoints

## Verification

```bash
# Before setup
curl http://frogswork.local/api/setup/status

# Run setup
curl -X POST http://frogswork.local/api/setup \
  -H "Content-Type: application/json" \
  -d '{"password":"test-admin-pass","timezone":"UTC"}'

# Login
curl -X POST http://frogswork.local/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"password":"test-admin-pass"}'

# Confirm folders on disk
ls /data/shared/
```

# M3 — Users + Samba sync

**Status:** Not started  
**Validates:** File user can SMB-authenticate; private folder visible only to that user (and super-users).

## Goal

CRUD for file users with sync to Linux accounts, Samba passdb, private folders under `/data/users/`, optional storage quotas, and super-user role.

## Prerequisites

- [M2](M2.md) complete
- Samba installed and base `smb.conf` deployed (M1)
- Confirm with user: super-user access on others' private folders is **read-only** (default)

## Implementation steps

### 1. Database extensions

- `file_users` — username, display_name, password_hash (or store Samba-sync only — prefer sync-on-write, not duplicate secrets in DB if avoidable; document choice)
- `is_superuser` boolean
- `quota_bytes` nullable

### 2. Integration: `linux_users.py`

- `create_user(username, home_path)` — `useradd`, home `/data/users/{username}`
- `delete_user(username)` — `userdel`, remove home dir
- `set_superuser(username, enabled)` — add/remove from group `frogswork-superuser`
- `set_quota(username, bytes)` — btrfs qgroups or `xfs_quota` equivalent on btrfs

### 3. Integration: `samba.py`

- `set_password(username, password)` — `smbpasswd -a`
- `delete_user(username)` — `smbpasswd -x` + remove
- Deploy base smb.conf from template if not already
- `[homes]` share for private folders

### 4. Super-user ACLs

- On super-user enable: grant read ACL on `/data/users/*` for `frogswork-superuser` group
- Private folder default: owner-only + super-user read

### 5. API routes (`frogswork_api/users/`)

| Endpoint | Purpose |
|----------|---------|
| `GET /api/users` | List file users |
| `POST /api/users` | Create user + sync |
| `GET /api/users/{id}` | Detail |
| `PATCH /api/users/{id}` | Password, quota, super-user |
| `DELETE /api/users/{id}` | Remove user + sync |

**Transaction strategy:** if Samba/Linux step fails after DB write, roll back DB row and log error with plain-language message.

### 6. Tests

- Unit tests with mocked integrations
- **NUC integration test:** create user → `smbclient -L //localhost -U user` or Windows Explorer login

# M1 — Provisioning

**Status:** Not started  
**Validates:** Fresh NUC serves `http://frogswork.local` and `GET /api/health` returns OK.

## Goal

Implement idempotent install scripts that provision a real appliance stack on the NUC: packages, btrfs `/data`, nginx, Avahi, systemd, SSH hardening defaults, and a minimal health API.

## Prerequisites

- [M0](M0.md) complete
- Asus NUC with **Ubuntu Server 24.04 LTS** installed (UEFI, OpenSSH enabled)
- SSH access from Windows laptop (`ssh user@<nuc-ip>`)
- Hostname set: `sudo hostnamectl set-hostname frogswork`

## Implementation steps

### 1. `scripts/install/00-prereqs.sh`

Install apt packages:

- `python3-venv`, `python3-pip`
- `nginx`
- `samba`, `samba-common-bin`
- `btrfs-progs`, `acl`
- `avahi-daemon`, `avahi-utils`
- `rsync` (for dev sync)

Make script idempotent (`apt-get install` is safe to re-run). Print package list before installing.

### 2. `scripts/install/01-partition.sh`

- Detect if `/data` btrfs mount already exists — **exit 0 if present** (idempotent guard)
- If not: create partition `p3` on NVMe (~440 GB), format btrfs, add to `/etc/fstab`
- Create subvolume layout under `/data`:
  - `users/`
  - `shared/`
  - `.snapshots/`
- Support `--dry-run` flag that prints planned actions without writing
- **Never wipe** the OS partition (`p2`)

### 3. `scripts/install/02-deploy-app.sh`

- Create system user `frogswork` (service account)
- Clone or rsync repo to `/opt/frogswork/`
- Python venv at `/opt/frogswork/venv`; `pip install` backend
- `npm ci && npm run build` in dashboard; output to `/opt/frogswork/dashboard/dist`
- Copy `deploy/nginx/frogswork.conf` → `/etc/nginx/sites-available/frogswork`; enable site; disable default
- Copy `deploy/systemd/*.service` and `*.timer` → `/etc/systemd/system/`
- Copy `deploy/avahi/frogswork.service` → `/etc/avahi/services/`
- Create `/var/lib/frogswork/` (state dir)
- `systemctl daemon-reload`

### 4. `scripts/install/03-enable-services.sh`

- `systemctl enable --now frogswork-api nginx avahi-daemon`
- Enable `frogswork-snapshot.timer` (timer only; snapshot scripts still placeholder until M6)
- SSH hardening defaults:
  - Document that password auth may stay on for initial dev if needed
  - Prepare for M7 dashboard toggle; for M1, document manual `sshd_config` posture
- Print status summary of enabled units

### 5. `scripts/dev/sync.sh`

- Accept `FROGSWORK_HOST` (default `frogswork.local`)
- `rsync` repo to NUC `/opt/frogswork/` excluding `.git`, `node_modules`, `.venv`, `dist`
- Remotely invoke `02-deploy-app.sh` via SSH

### 6. Health endpoint

In [`backend/frogswork_api/main.py`](../../backend/frogswork_api/main.py):

    return {"status": "ok", "version": "..."}

Read version from package or `VERSION` file.

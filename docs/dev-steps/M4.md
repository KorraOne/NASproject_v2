# M4 — Folders + ACLs

**Status:** Complete  
**Validates:** Explorer access respects read-only vs read-write per user per shared folder; super-user can read all private folders.

## Goal

Shared folder permission matrix (user × folder × none | read | read-write), dynamic Samba share configs, and POSIX ACL sync.

## Prerequisites

- [M3](M3.md) complete — file users and Samba auth working
- Seeded folders from setup exist under `/data/shared/`

## Implementation steps

### 1. Database

- `folder_permissions` — `file_user_id`, `shared_folder_id`, `access` enum (`read`, `read_write`)
- Unique constraint on (user, folder)

### 2. Integration: ACL sync

- For each shared folder `/data/shared/{name}`:
  - Write `/etc/samba/shares.d/{name}.conf` share section
  - Apply `setfacl` for each user with access
  - `read` → `r-x`; `read_write` → `rwx`
- After changes: `testparm` && `systemctl reload smbd`

### 3. Samba share naming

- Share name pattern: `shared-Projects` → `\\frogswork.local\shared-Projects`
- Document mapping for helper app (M8)

### 4. API routes (`frogswork_api/folders/`)

| Endpoint | Purpose |
|----------|---------|
| `GET /api/folders` | List shared folders + permission summary |
| `POST /api/folders` | Create folder (disk + DB + smb fragment) |
| `PATCH /api/folders/{id}` | Rename, bulk permission update |
| `DELETE /api/folders/{id}` | Remove folder (guard if not empty?) |
| `PUT /api/folders/{id}/permissions` | Replace permission matrix for folder |

Users page should use same service layer (single source of truth).

### 5. Tests

- Unit: permission matrix logic
- NUC: two users, one folder read-only for alice, read-write for bob — verify in Explorer

## Verification

1. Create users alice and bob
2. Assign Projects: alice read-only, bob read-write
3. From Windows as bob: create file in Projects — success
4. From Windows as alice: try delete bob's file — denied
5. Super-user can open `\\frogswork.local\homes` paths for all users

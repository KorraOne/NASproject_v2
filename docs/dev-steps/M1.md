# M1 — Provisioning

**Status:** Complete  
**Validates:** NUC serves `GET /api/health` on LAN; `/data` btrfs mounted; nginx, API, Samba, Avahi active.

## Goal

Implement idempotent install scripts that provision a real appliance stack on the NUC: packages, btrfs `/data`, nginx, Avahi, systemd, and a minimal health API.

## Prerequisites

- [M0](M0.md) complete
- Asus NUC with **Ubuntu Server 26.04 LTS** (tested; default LVM installer layout)
- SSH access (`ssh nuc1`) with passwordless sudo
- Hostname `frogswork`

## Implementation steps

### 1. `scripts/install/00-prereqs.sh`

- [x] apt: python3-venv, nginx, samba, btrfs-progs, acl, avahi, rsync, nodejs, npm, lvm2

### 2. `scripts/install/01-partition.sh`

- [x] Idempotent guard if `/data` already mounted
- [x] Create LV `ubuntu-vg/frogswork-data` (90% free space) — **not raw repartition**
- [x] mkfs.btrfs, fstab, mount `/data`
- [x] Directories: `users/`, `shared/`, `.snapshots/`
- [x] `--dry-run` support

### 3. `scripts/install/02-deploy-app.sh`

- [x] `frogswork` system user, `/var/lib/frogswork`
- [x] Python venv + editable backend install
- [x] `npm ci && npm run build` for dashboard
- [x] nginx, systemd, avahi, samba base config
- [x] Deploy user (`korra`) ownership for dev sync

### 4. `scripts/install/03-enable-services.sh`

- [x] Enable/start frogswork-api, nginx, avahi, smbd
- [x] SSH left open for development (M7 adds toggle)

### 5. `scripts/install/install.sh`

- [x] Runs 00 → 03 in order

### 6. `scripts/dev/sync.sh`

- [x] rsync or tar-over-SSH (Windows-friendly)
- [x] CRLF normalisation for shell scripts
- [x] Re-runs 02 + 03 after sync

### 7. Health endpoint

- [x] `GET /api/health` → `{"status":"ok","version":"..."}`
- [x] [`backend/frogswork_api/paths.py`](../../backend/frogswork_api/paths.py)

### 8. Deploy config fixes

- [x] API on `127.0.0.1:8000` (not unix socket — avoids nginx permission issues)
- [x] Samba `workgroup` / `netbios name` fix

## Verification (NUC — 2026-07-04)

```bash
curl http://192.168.0.232/api/health
# {"status":"ok","version":"0.0.0-dev"}

systemctl is-active frogswork-api nginx avahi-daemon smbd
# active active active active

df -h /data
# ~310G btrfs on /dev/ubuntu-vg/frogswork-data
```

## Deviations from original plan

| Plan | Actual |
|------|--------|
| Ubuntu 24.04 | **26.04 LTS** — works |
| Raw `p3` btrfs partition | **LVM LV** from installer free space |
| uvicorn unix socket | **127.0.0.1:8000** + nginx proxy |
| rsync from Windows | **scp / tar** fallback in sync.sh |

# M6 — Snapshots

**Status:** Complete  
**Validates:** Delete a file → restore from last nightly snapshot via dashboard → file reappears in Explorer.

## Goal

btrfs snapshot scripts, systemd timer, retention policy, list/create/restore API, and dashboard Snapshots page.

## Prerequisites

- [M1](M1.md) btrfs `/data` layout
- [M5](M5.md) dashboard shell (add Snapshots page)

## Implementation steps

### 1. `scripts/btrfs/snapshot.sh`

- `btrfs subvolume snapshot -r /data /data/.snapshots/daily-YYYY-MM-DD`
- Skip if today's daily already exists
- Exit non-zero on failure with clear log message

### 2. `scripts/btrfs/apply-retention.sh`

- Read policy from SQLite or config file (defaults: 7 daily, 4 weekly, 3 monthly)
- Promote dailies to weeklies/monthlies per rules
- Delete expired readonly subvolumes under `/data/.snapshots/`

### 3. Integration: `btrfs.py`

- `list_snapshots()` — parse subvolume list
- `restore_path(snapshot, source_rel, dest_rel)` — copy from snapshot subvol to live path
- Guard restore with confirmation token in API

### 4. API (`frogswork_api/snapshots/`)

| Endpoint | Purpose |
|----------|---------|
| `GET /api/snapshots` | List snapshots with dates and types |
| `POST /api/snapshots` | Manual snapshot now |
| `GET /api/snapshots/settings` | Schedule + retention |
| `PATCH /api/snapshots/settings` | Update schedule (default 02:00) and retention |
| `POST /api/snapshots/{id}/restore` | Restore file or folder path |

### 5. systemd

- Wire `frogswork-snapshot.service` to real scripts
- Enable `frogswork-snapshot.timer` (02:00 nightly)

### 6. Dashboard

- **Snapshots** page: list, create now, browse paths, restore with confirm dialog
- Settings section for schedule/retention

## Verification

```bash
# Manual snapshot
sudo /opt/frogswork/scripts/btrfs/snapshot.sh
btrfs subvolume list /data

# Timer
systemctl list-timers frogswork-snapshot.timer

# E2E: create file → snapshot → delete file → restore via dashboard → file back
```

# M3 — Users + Samba sync

**Status:** Complete  
**Validates:** File user created via API; Linux account, private folder, Samba passdb entry.

## Goal

CRUD for file users with sync to Linux accounts, Samba passdb, private folders under `/data/users/`, optional storage quotas, and super-user role.

## Prerequisites

- [M2](M2.md) complete
- Samba installed; `deploy/sudoers/frogswork-integrations` allows `frogswork` user to run integration commands

## Implementation steps

### 1. Database

- [x] `file_users` table (no password in DB — Samba passdb only)

### 2. Integrations

- [x] `integrations/_run.py` — sudo wrapper with `FROGSWORK_SKIP_SYSTEM=1` for tests
- [x] `linux_users.py` — useradd, btrfs home subvolume, ACLs, super-user group, quota
- [x] `samba.py` — smbpasswd, reload smbd

### 3. Service layer

- [x] `services/users.py` — create/update/delete with rollback on integration failure

### 4. API

- [x] `GET/POST /api/users`, `GET/PATCH/DELETE /api/users/{id}` (admin auth required)

### 5. Deploy

- [x] `deploy/sudoers/frogswork-integrations`
- [x] `chmod a+rX` on backend after deploy (frogswork user must import modules)

### 6. Tests

- [x] 16 pytest tests including `test_users.py`
